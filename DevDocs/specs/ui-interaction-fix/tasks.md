## 任务详情

### T-001 分析当前 UI 层级结构
**描述：** 梳理 Combat.tscn 的节点树，标记每个 Control 的 mouse_filter 设置
**步骤：**
1. 导出 Combat.tscn 的节点树
2. 记录每个 Control 节点的 MouseFilter 值
3. 绘制事件传递流程图
4. 标识问题节点
**验收标准：** 产出包含节点树和 mouse_filter 值的文档

### T-002 修复 CombatUI 的 mouse_filter 配置
**描述：** 将 CombatUI 的 mouse_filter 改为 Ignore 或 Pass，确保不拦截底层事件
**步骤：**
1. 确认 CombatUI 本身不需要接收鼠标事件
2. 将 MouseFilter 改为 Ignore
3. 确保子节点（如 DeckViewUI）需要时设置 Stop
**验收标准：** EndTurnButton、牌堆按钮可正常点击

### T-003 优化 CardUI 拖拽状态管理
**描述：** 确保拖拽状态切换清晰，避免状态残留
**步骤：**
1. 检查 IsDragging 状态的设置和清除时机
2. 确保 EndDrag 在所有情况下都被调用
3. 添加状态断言检查
**验收标准：** 拖拽开始和结束状态清晰，无状态残留

### T-004 优化 HandUI 拖拽层事件传递
**描述：** 确保 DragLayer 正确设置 MouseFilter，允许事件穿透到目标
**步骤：**
1. 确认 DragLayer 的 MouseFilter=Ignore
2. 确保拖拽中的卡牌不会阻挡目标检测
3. 验证目标检测使用正确的坐标系
**验收标准：** 拖拽时可正确检测地图节点和敌人

### T-005 添加 SelectionMode 状态日志
**描述：** 在状态切换时添加详细日志
**步骤：**
1. 在 SetSelectionMode 方法中添加日志
2. 记录旧状态和新状态
3. 记录触发原因
**验收标准：** 每次状态切换有清晰日志

### T-006 增强事件处理日志
**描述：** 在关键事件处理点添加日志
**步骤：**
1. CardUI: 点击、拖拽开始/结束
2. HandUI: 拖拽层管理
3. CombatUI: 目标检测
4. 统一日志格式
**验收标准：** 所有关键事件有日志

### T-007 测试所有交互场景
**描述：** 执行手动测试覆盖所有交互场景
**步骤：**
1. 测试点击 EndTurnButton
2. 测试点击牌堆按钮
3. 测试卡牌拖拽部署
4. 测试卡牌拖拽攻击敌人
5. 测试无效目标拖拽
6. 测试状态切换
**验收标准：** 所有场景通过验收清单
