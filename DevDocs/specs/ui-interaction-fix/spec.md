# Spec: UI 交互修复

> 状态: DRAFT
> 创建: 2026-03-06 00:45
> 更新: 2026-03-06 00:45

## 背景与目标

**背景：**
OdysseyCards 战斗界面的 `CombatUI` 控件设置了 `mouse_filter=Stop`，导致所有点击事件被拦截，无法操作底层的按钮（如 EndTurnButton）。改为 `Pass` 后点击可以穿透，但交互逻辑变得混乱，表现为：
- 卡牌拖拽和放置目标检测不一致
- 选择模式（SelectionMode）状态切换不可预测
- 多个 UI 层的事件处理顺序冲突

**目标：**
修复 UI 交互问题，确保：
1. 所有按钮可正常点击
2. 卡牌拖拽、放置、选择行为正确
3. 选择模式状态机工作正常
4. UI 层级事件传递清晰可控

## 范围

**In：**
- CombatUI 的 mouse_filter 配置
- CardUI 的输入处理逻辑
- HandUI 的拖拽管理
- CombatManager 的 SelectionMode 状态机
- 事件传递顺序优化

**Out：**
- 游戏逻辑改动（卡牌效果、伤害计算等）
- UI 视觉样式改动
- 新功能开发

## 用户场景

### 场景1：结束回合
**角色：** 玩家
**目标：** 点击"End Turn"按钮结束回合
**流程：**
1. 玩家点击屏幕右下角"End Turn"按钮
2. 按钮响应点击事件
3. 回合结束，敌人回合开始

### 场景2：部署单位
**角色：** 玩家
**目标：** 从手牌中拖拽单位卡牌到地图节点部署
**流程：**
1. 玩家拖拽单位卡牌
2. 卡牌跟随鼠标移动
3. 悬停在地图像素上时高亮显示有效目标
4. 释放时检测目标节点
5. 如果是有效部署点，执行部署
6. 否则卡牌返回手牌

### 场景3：对敌人使用指令卡
**角色：** 玩家
**目标：** 将指令卡拖拽到敌人身上使用
**流程：**
1. 玩家拖拽指令卡牌
2. 悬停在敌人上时高亮
3. 释放时检测目标敌人
4. 如果是指令的有效目标，执行效果
5. 否则卡牌返回手牌

## 功能需求

### REQ-001 CombatUI 事件传递配置
**描述：** CombatUI 应允许底层 UI 接收点击事件
**验收标准：**
- EndTurnButton 可正常点击
- DrawPileButton 可正常点击
- DiscardPileButton 可正常点击
- 地图节点可正常点击
**优先级：** P0

### REQ-002 CardUI 拖拽状态管理
**描述：** CardUI 的拖拽状态应清晰且不与其他状态冲突
**验收标准：**
- 拖拽开始时正确记录原始位置
- 拖拽过程中卡牌跟随鼠标
- 拖拽结束时正确检测目标
- 无效目标时卡牌返回原位
**优先级：** P0

### REQ-003 HandUI 拖拽层管理
**描述：** HandUI 的拖拽层应独立于其他 UI 层
**验收标准：**
- 拖拽中的卡牌在最上层显示
- 拖拽层不干扰其他 UI 元素
- 拖拽层正确转发事件
**优先级：** P0

### REQ-004 SelectionMode 状态机
**描述：** CombatManager 的 SelectionMode 应与 UI 状态同步
**验收标准：**
- 部署单位时进入 DeployUnit 模式
- 移动单位时进入 MoveUnit 模式
- 攻击时进入 AttackTarget 模式
- 无操作时为 None 模式
- 状态切换有清晰的日志
**优先级：** P0

### REQ-005 事件处理日志
**描述：** 关键 UI 事件应有足够的日志用于调试
**验收标准：**
- 点击事件有日志
- 拖拽事件有日志
- 状态切换有日志
- 目标检测有日志
**优先级：** P1

## 非功能需求

**性能：**
- UI 交互响应时间 < 16ms
- 拖拽无卡顿

**可维护性：**
- 事件处理逻辑集中在各自的类中
- 避免循环依赖

**可测试性：**
- 关键方法可单独测试
- 状态可查询

## 约束与依赖

**约束：**
- 不改变现有游戏逻辑
- 保持 Godot 4.x 兼容

**依赖：**
- Godot 的输入事件系统
- 现有的 CombatManager 状态机

## 风险

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| 事件传递顺序复杂 | 中 | 高 | 使用 ZIndex 和 MouseFilter 明确层级 |
| 拖拽层与目标检测冲突 | 中 | 中 | 确保 DragLayer 的 MouseFilter=Ignore |
| 状态机并发问题 | 低 | 高 | 添加状态锁或队列 |
